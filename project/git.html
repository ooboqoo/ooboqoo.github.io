龙猫关于多人合作项目的经验分享

  1. 多用客户端和工具，少用命令行，除非是在 linux 服务器上直接开发。
  2. 每次提交前，diff 自己的代码，以免提交错误代码或测试代码。
  3. 下班回家前，整理好自己的工作区并提交。
  4. 并行的项目，使用分支开发。
  5. 遇到冲突时，搞明白冲突的原因，千万不要随意丢弃别人的代码。
  6. 产品发布后，记得打 tag，方便将来拉分支修 bug。
